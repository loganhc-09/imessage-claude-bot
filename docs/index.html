iMessage Claude Bot

Built by @loganinthefuture with Claude Code

A Python script that turns your iMessage self-chat into a full AI agent. Text yourself from your phone — Claude Code responds in the same thread. File access, web search, code execution, image understanding. No apps, no APIs, no monthly fees.

How it works: Your Mac stores every iMessage in a local SQLite database. This script watches that database, routes your messages to Claude Code CLI, and sends responses back via AppleScript. Everything runs locally.

Setup

About 10 minutes. No dependencies beyond Python 3 (ships with macOS) and Claude Code CLI.

1
Grant Full Disk Access to Python

The bot reads your Messages database, which macOS protects. Open System Settings → Privacy & Security → Full Disk Access, click +, press Cmd+Shift+G, type /usr/bin/python3, and add it.

2
Clone and configure
git clone https://github.com/loganhc-09/imessage-claude-bot.git
cd imessage-claude-bot
cp .env.example .env

Edit .env with your details:

APPLE_ID=you@icloud.com
PHONE_NUMBER=+15551234567
CLAUDE_WORKSPACE=/path/to/your/project
CLAUDE_PATH=/usr/local/bin/claude
3
Create your self-chat

Open Messages on your phone and text yourself via iMessage. This creates the thread the bot monitors. You only need to do this once.

4
Run
python3 imessage-bot.py

Text yourself from your phone. Claude responds in the same thread.

Run permanently

The repo includes a launchd plist template to keep the bot running in the background:

# Edit the plist with your paths first
cp com.claude.imessage-bot.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.claude.imessage-bot.plist

# Check logs
tail -f /tmp/imessage-bot.log

Commands

/new
Start a fresh Claude session
/status
Check if the bot is running

Gotchas

We hit every one of these building this. Saving you the debugging.

Two self-chats

iMessage creates separate threads for your email and phone number. Your text might land in one while the bot watches the other. That's why .env has both APPLE_ID and PHONE_NUMBER — the bot monitors both and replies to whichever you messaged from.

Full Disk Access after restart

When launchd restarts Python, FDA can take seconds to activate. The bot retries automatically for up to 2.5 minutes. If you see "authorization denied" in the error log, it usually resolves itself.

The attributedBody problem

Messages sent via AppleScript store text in a binary blob (NSAttributedString typedstream) instead of the text column. The bot includes a custom parser for Apple's undocumented format. If messages seem invisible, this is probably why.

Image attachments need staging

Images live in ~/Library/Messages/Attachments/ which requires FDA. The bot's Python process has FDA, but Claude CLI doesn't. So the bot copies images to /tmp/ before passing paths to Claude. This is automatic.

iCloud sync delay

Messages from your phone sync to your Mac via iCloud — usually instant, sometimes 30-60 seconds. If the bot doesn't respond, make sure Messages on your Mac is open and syncing.

Infinite loop prevention

You're texting yourself, so the bot's replies appear as new messages. Without protection, it would process its own responses forever. We prevent this with MD5 hash tracking of sent messages and ROWID advancement after each send.

Under the hood

1
Poll — Reads ~/Library/Messages/chat.db every 3 seconds for new messages in your self-chat
2
Extract — Pulls text from either the text column or the attributedBody binary blob
3
Stage — Copies any image attachments to /tmp/ where Claude CLI can read them
4
Process — Sends the message to Claude Code CLI with session persistence via --resume
5
Reply — Sends Claude's response back via AppleScript to the same thread
6
Advance — Skips past its own reply in the database to prevent infinite loops

No network APIs. No third-party services. Everything runs locally on your Mac.

Make it yours

Give Claude context

Create a CLAUDE.md file in your workspace. The bot reads this on every new session. Put your preferences, projects, role, tone — whatever you want Claude to know about you.

Log conversations

Set LOG_DIR in .env to save daily conversation logs as markdown. Useful for building memory systems or reviewing what you've discussed.

Chief of staff mode

With a good CLAUDE.md, Claude can act as a personal assistant — morning briefings, meeting prep, follow-up tracking, research. The interface is just texting.
