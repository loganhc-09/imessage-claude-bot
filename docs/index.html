This is a Python script that bridges your iMessage self-chat to Claude Code CLI. It watches your Mac's Messages database, sends your texts to Claude, and replies back in the same thread.

You get a full AI agent from your phone — file access, web search, code execution, image understanding — in the app you already use.

Setup (10 minutes)

Prerequisites

  • Mac running macOS 13 (Ventura) or later
  • Claude Code CLI installed (npm install -g @anthropic-ai/claude-code)
  • An active Claude Code subscription

1 Grant Full Disk Access to Python

The bot needs to read your Messages database. Open:

System Settings → Privacy & Security → Full Disk Access

Click +, press Cmd+Shift+G, type /usr/bin/python3, and add it.

2 Clone and configure

git clone https://github.com/loganhc-09/imessage-claude-bot.git
cd imessage-claude-bot
cp .env.example .env

Edit .env with your Apple ID and phone number:

APPLE_ID=you@icloud.com
PHONE_NUMBER=+15551234567
CLAUDE_WORKSPACE=/path/to/your/project
CLAUDE_PATH=/usr/local/bin/claude

3 Create your self-chat

Open Messages on your phone and send yourself a text via iMessage. This creates the thread the bot monitors.

4 Run it

python3 imessage-bot.py

Text yourself from your phone. Claude responds in the same thread.

Run permanently (launchd)

To keep the bot running in the background:

# Edit the plist with your paths first
cp com.claude.imessage-bot.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.claude.imessage-bot.plist

# Check logs
tail -f /tmp/imessage-bot.log

Commands

  • /new — Start a fresh Claude session
  • /status — Check if the bot is running

Gotchas (the hard-won knowledge)

These are the problems we hit building this. You'll probably hit them too.

⚠️ Two self-chats

iMessage creates separate threads for your email and phone number. If you text yourself from your phone, it might land in the phone-number thread while the bot watches the email thread. That's why .env has both APPLE_ID and PHONE_NUMBER — the bot monitors both and replies to whichever one you messaged from.

⚠️ Full Disk Access after restart

When launchd restarts the Python process, FDA can take a few seconds to kick in. The bot retries automatically for up to 2.5 minutes. If you see "authorization denied" in the error log, it usually resolves itself. If it doesn't, re-add /usr/bin/python3 to FDA.

⚠️ attributedBody parsing

Messages sent via AppleScript store text in a binary blob (NSAttributedString "typedstream" format) instead of the text column. The bot includes a parser for this. If you see empty messages being skipped, this is likely why — the parser handles the common cases but Apple's format is undocumented.

⚠️ Image attachments need staging

Images are stored in ~/Library/Messages/Attachments/, which requires FDA. The bot (python3) has FDA, but Claude Code CLI doesn't. So the bot copies images to /tmp/ before passing paths to Claude. This happens automatically.

⚠️ iCloud sync delay

Messages sent from your phone sync to your Mac via iCloud. This is usually instant, but can take 30-60 seconds. If the bot doesn't respond, check that the Messages app on your Mac is open and syncing. Having it in the foreground helps.

⚠️ Infinite loop prevention

Since you're texting yourself, the bot's replies appear as new messages. Without protection, the bot would process its own responses forever. We prevent this with MD5 hash tracking of sent messages and ROWID advancement after each send.

How it works

Your Mac stores every iMessage in a SQLite database at ~/Library/Messages/chat.db. The bot:

  1. Polls chat.db every 3 seconds for new messages in your self-chat
  2. Extracts text (from either the text column or the attributedBody binary blob)
  3. Copies any image attachments to /tmp/ for Claude to read
  4. Sends the message to Claude Code CLI (claude -p) with session persistence
  5. Replies via AppleScript to the same thread the message came from
  6. Advances past its own reply to prevent infinite loops

No network APIs. No third-party services. Everything runs locally on your Mac.

Make it yours

Give Claude context about you

Create a CLAUDE.md file in your workspace directory. The bot reads this on every new session. Put your preferences, projects, personality notes — whatever you want Claude to know about you.

Log conversations

Set LOG_DIR in .env to save daily conversation logs as markdown files. Useful for building a memory system or reviewing what you've discussed.

Use as a chief of staff

With a good CLAUDE.md, Claude can serve as a personal assistant — morning briefings, meeting prep, follow-up tracking, content triage. The interface is just texting.
